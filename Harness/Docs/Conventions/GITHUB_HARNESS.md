## 커밋 메시지

커밋 메시지는 한국어로 작성한다.

프로젝트 규칙이 추가로 정해지기 전까지는 짧고 명확한 한국어 메시지를 사용한다.

예:

- `docs: AI 개발 하네스 추가`
- `fix: 플레이어 이동 범위 보정`
- `feat: 플레이어 액션 추가`

## 커밋 타입

- `feat`: 기능 추가
- `fix`: 버그 수정
- `docs`: 문서 변경
- `refactor`: 동작 변경 없는 구조 개선
- `test`: 테스트 또는 검증 코드
- `chore`: 빌드, 설정, 기타 작업

## 작업 흐름

1. 작업 전 `git status`를 확인한다.
2. 기존 수정사항과 작업 범위가 충돌하는지 확인한다.
3. 관련 파일을 읽고 수정한다.
4. 가능한 검증을 수행한다.
5. `FILE_CHANGE_LOG.md`를 갱신한다.
6. 커밋은 사용자가 요청한 경우에만 진행하고, 메시지는 한국어로 작성한다.
7. PR은 사용자가 요청한 경우에만 진행한다.
